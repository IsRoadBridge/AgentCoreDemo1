# github远程连不上
1. 打开Windows「设置」→「网络和 Internet」→「代理」。
找到「使用代理服务器」部分，记录下“端口”后面的数字
2. 仅针对GitHub配置代理
git config --global http.https://github.com.proxy http://127.0.0.1:<你的代理端口>
3. 清除本地DNS缓存：防止旧的DNS解析记录干扰。
windows   本地命令行执行 ipconfig /flushdns
# 4.  如果还有问题 更换Git版本，降低HTTP协议：某些网络环境中HTTP/2协议可能不稳定，可尝试强制降级到HTTP/1.1（备用）
# git config --global http.version HTTP/1.1
# 5. 不使用VPN时，清除Git的代理设置
# git config --global --unset http.proxy
# git config --global --unset https.proxy
